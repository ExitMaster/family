# 연어항해일지 v2 — 개발 지시서

2026년 7월 전면 개정판. 기존 "일기 아카이빙 + AI 심리상담" 기획을 대체한다.

## 1. 프로젝트 개요

ADHD 사용자의 작업기억을 보조하는 **브레인 덤프 + AI 트리아지** 웹앱.

- **사용 목적 (1)**: 떠오르는 아이디어를 마찰 없이 빠르게 덤프하고, 하던 과제에 즉시 복귀할 수 있게 한다.
- **사용 목적 (2)**: 수많은 아이디어 사이의 우선순위를 정리하고 실행으로 옮기는 것을 돕는다.
- 기존 일기 아카이빙·심리상담(카운슬러)·레퍼런스 DB 기능은 **부기능으로 축소 유지**한다.
- 개발자 1인(Claude Code) 단독 구현, 배포 대상 본인 1인(ku.hayoungkim@gmail.com — Firebase 소유 계정과 동일). 모바일(갤럭시 Z 폴드 7) 중심, PC 및 다기기 접근 지원.

### 디자인 원칙

- **극도로 미니멀.** 화려한 연출은 인지부하를 늘리므로 배제한다. 완료 처리 시 취소선/페이드 등 최소한의 피드백만 사용.
- **필터링은 엄격하게, 아카이브는 직관적으로.** 실행 화면(포커스 모드)에서는 한 번에 하나만 보여주고, 보관된 데이터는 필요할 때 바로 찾을 수 있게 한다.
- **캡처는 3초 컷.** 앱 실행 → 입력 → 저장까지의 마찰을 최소화한다. 완벽하게 적지 않아도 일단 던져둘 수 있어야 한다.
- **죄책감 없는 이월.** 미완료 항목에 빨간 경고·지연 표시를 하지 않는다. 부드러운 톤으로 재검토를 유도한다.

## 2. 기술 스택 및 아키텍처

| 역할 | 기술 |
|---|---|
| 프론트엔드 | React + Vite |
| 소스 관리 | GitHub |
| 배포 + 서버리스 함수 | Vercel (GitHub push 시 자동 배포) |
| 인증 | Firebase Google Auth |
| 데이터베이스 | Firebase Firestore (오프라인 persistence 활성화) |
| 파일 저장 | Firebase Storage (부기능용) |
| AI | Claude API — 모델명은 설정 파일 한 곳에서 관리 (기본: claude-sonnet-4-6) |
| OCR (부기능) | Google Cloud Vision API |
| PWA | vite-plugin-pwa |

### 아키텍처 원칙

- **Claude API·Vision API는 클라이언트에서 직접 호출하지 않는다.** 반드시 Vercel Serverless Function을 프록시로 경유한다. API 키는 서버 환경변수에만 존재하며, 함수 진입 시 Firebase ID 토큰을 검증해 허용된 이메일만 통과시킨다.
- **토큰 비용 절감 원칙**: 인박스 분류 제안은 저녁 정리 직전 일괄(batch) 1회 호출로 처리. 우선순위 재계산·할일 쪼개기·아이디어 리뷰·URL 요약·카운슬러 분석은 모두 사용자가 버튼을 눌러야만 실행.

## 3. 데이터 구조

### 3.1 카테고리 (5종 + 인박스)

| 카테고리 | 정의 | 기한 | 이월/리뷰 |
|---|---|---|---|
| **프로젝트** | 장기 전략 구상 → 할일 체계화 → task 설계 → 실행 (이벤트성 행사 준비 포함) | 선택 | 하위 할일 단위로 이월 |
| **할일** | 1건짜리 실행 과업 | 선택 | O (죄책감 없는 이월) |
| **쇼핑** | 구매 목록. 태그 없음, 독립 리스트 | 없음 | 체크만 |
| **아이디어** | 스치는 생각. 실행과 직결되지 않으며 디벨롭 필요 | 없음 | 주간 리뷰 → 할일/프로젝트로 승격 또는 폐기 |
| **생각** | 순수 기록·일기·단상. 실행으로 이어질 필요 없음 | 없음 | 없음 (아카이브) |
| (인박스) | 미분류 상태. 카테고리·태그를 아무것도 선택하지 않고 저장한 항목 | — | 저녁 정리 시 일괄 처리 |

### 3.2 태그 체계 (2계층)

- **주제 태그** — `#교육` 형식. 고정 6종: `#일기` `#심리` `#교육` `#의료` `#일` `#금융`. 카운슬러·레퍼런스 부기능에서 사용.
- **프로젝트 태그** — `.논문작업` 형식. 프로젝트 생성 시 자동 발급. 할일·아이디어·생각을 프로젝트에 귀속시키는 수단 (별도 parentId 없음).
- 태그 바 UI: 프로젝트 태그 구간 / 주제 태그 구간을 나누어 표시. 최근 사용 태그만 노출하고 "더보기" 드롭다운으로 전체 접근. 프로젝트 태그 구간은 `projects` 컬렉션의 active 항목만 렌더링한다.

### 3.3 Firestore 컬렉션

**`entries`**

| 필드 | 타입 | 설명 |
|---|---|---|
| id | string | 자동 생성 문서 ID |
| content | string | 원문 텍스트 |
| category | string | 'project' / 'task' / 'shopping' / 'idea' / 'note' / 'inbox' |
| suggestedCategory | string \| null | AI가 추정한 카테고리 (인박스 항목의 제안 라벨) |
| status | string | 'open' / 'done' / 'migrated' / 'discarded' / 'archived' |
| priority | number \| null | AI 정렬 순번 (1이 최상위). "정리하기" 실행 시에만 갱신 |
| pinned | boolean | 수동 고정. true인 항목은 AI가 순위를 바꾸지 않음 |
| dueDate | timestamp \| null | 기한 (할일·프로젝트만 사용) |
| tags | array\<string\> | 주제 태그 + 프로젝트 태그 혼용 |
| migratedCount | number | 이월 횟수 |
| skipCount | number | 포커스 모드 건너뛰기 횟수 |
| classifiedBy | string | 'auto' / 'manual' |
| createdAt / updatedAt | timestamp | 생성·수정 일시 |

**`projects`**

| 필드 | 타입 | 설명 |
|---|---|---|
| id | string | 자동 생성 문서 ID |
| name | string | 프로젝트명 |
| tagName | string | 자동 발급 태그 (예: '.논문작업') |
| goal | string | 목표 (프로젝트 상세 설정에서 입력) |
| dueDate | timestamp \| null | 전체 기한 |
| status | string | 'active' / 'paused' / 'done' |
| createdAt | timestamp | 생성 일시 |

**`settings`** (단일 문서)

| 필드 | 타입 | 설명 |
|---|---|---|
| eveningReviewHour | number | 저녁 정리 노출 시각 (기본 20) |

## 4. 화면 구성

### 4.0 앱 전체 구조

- **하단 탭 3개**: [덤프] (캡처, 시작 화면) / [정리] (리스트·프로젝트·검색) / [포커스] (단일 카드 + 뽀모도로)
- **설정 하위 메뉴**: 카운슬러(부기능) / 레퍼런스(부기능) / 도파민 메뉴 관리 / 저녁 정리 시각 / 생체인증 / 사이드 버튼 설정 가이드

### 4.1 덤프 화면 (메인·시작 화면)

- 중앙: 깔끔한 텍스트 입력창 1개. 진입 즉시 커서 포커스. (음성 입력은 OS 키보드의 음성인식 기능 사용 — 별도 구현 없음)
- 좌측: 카테고리 아이콘 5종 세로 배치 (불렛저널 색인 스타일). 터치 시 해당 카테고리로 **즉시 확정 저장**.
- 우측: 기한 슬롯. 터치 시 달력 UI.
- 하단: 태그 바 (3.2 참조). 터치 시 태그 부착. 선택은 옵션.
- **아무것도 누르지 않고 저장** → `category: 'inbox'`로 저장. AI 분류는 즉시 실행하지 않고, 저녁 정리 직전 일괄 호출로 `suggestedCategory`를 채운다.
- [프로젝트] 버튼으로 저장한 경우 → 즉시 저장 후 **프로젝트 상세 설정 화면으로 자동 전환** (이름 확정, `.태그` 자동 발급, 목표·기한 입력).

### 4.2 정리 화면

- 카테고리별 필터, 텍스트 검색 (클라이언트 사이드 필터링 — Firestore에는 전문검색이 없으므로 로드된 항목을 프론트에서 검색).
- 인박스 badge 표시.
- **"정리하기" 버튼**: 미완료 항목 전체를 Claude API로 재분석 → 우선순위 재정렬. pinned 항목은 순위 유지. 순위 변경 근거를 한 줄씩 표시 (예: "기한이 이틀 남아 올렸어요"). skipCount 3회 이상 항목은 쪼개기/폐기 제안. migratedCount 3회 초과 항목도 쪼개기/폐기 제안.

### 4.3 저녁 정리 (통합 리뷰 화면)

푸시 알림 없이 **로컬 트리거**로 동작: 설정된 시각(기본 20시) 이후 앱을 열면 처리할 항목이 있을 경우 이 화면을 우선 노출.

1. **인박스 정리**: 각 항목에 AI 제안 라벨 표시 → 원탭 확정 또는 수정.
2. **할일 이월 리뷰**: 대상 = (a) 기한 지난 할일 + (b) 포커스 모드에 노출됐으나 미완료인 항목. 각 항목에 `>` (내일로) / `>>` (주말로) / 완료 / 폐기 선택지. 빨간 경고 금지. "내일 하면 되죠 — 오늘 이만큼 적어둔 것도 진전이에요" 톤의 카피 사용.
3. **아이디어 리뷰** (주간): 마지막 리뷰로부터 7일 경과 시 이 화면에 섹션으로 함께 노출. AI가 항목별로 할일 전환 / 프로젝트 승격 / 폐기 / 보류를 제안 → 사용자 확정. 승격 시 category 변경, 프로젝트 승격 시 프로젝트 상세 설정 흐름으로 연결.

### 4.4 포커스 모드

- 우선순위 1위 항목 **단 하나만** 큰 카드로 표시.
- 버튼 3개: **[완료]** / **[건너뛰기]** (다음 순위 카드 표시, skipCount +1, 죄책감 유발 문구 없음) / **[쪼개기]** (4.6 직결).
- 카드에서 바로 뽀모도로 타이머 시작 가능.

### 4.5 뽀모도로 타이머 (부가기능)

- 고정 패턴: **25분 집중 – 5분 휴식 – 25분 – 5분 – 25분 – 15분 긴 휴식**.
- 숫자 카운트다운이 아닌 **비주얼 타이머** (색 영역이 줄어드는 Time Timer 형태) — 시간감각(Time Blindness) 보완.
- 휴식 시작 시 도파민 메뉴에서 랜덤 1개 추천 노출 (설정에서 사용자가 목록 등록).

### 4.6 할일 쪼개기 마법사

- 할일 상세 및 포커스 카드에서 [쪼개기] 버튼 → Claude API 호출 → "시작할 수 있는 최소 단위" 3~4단계 하위 체크리스트 생성 → 사용자 확인 후 저장.

### 4.7 프로젝트 상세

- 해당 `.프로젝트태그`가 달린 모든 항목(할일/아이디어/생각) 모아보기 + 프로젝트 목표·기한·상태 관리.
- 프로젝트를 done/paused로 변경하면 태그 바에서 자동 제외.

### 4.8 카운슬러 / 레퍼런스 (부기능, 축소)

- 기존 기획의 기능을 유지하되 하단 탭에서 제외하고 설정 하위로 이동.
- 카운슬러: 기간·태그 선택 → 분석 실행(버튼 클릭 시에만 API 호출) → 현재 상태 요약 / 조언 / 액션플랜 / 생각해볼 주제 + 후속 채팅. 심리 마스터 요약본(psych_summary/master) 생성·갱신 버튼 포함.
- 레퍼런스: 사진 OCR(원본 미보관) / 유튜브 URL(자막 추출 후 요약) / 기사 URL(본문 크롤링 후 요약). 요약은 버튼 클릭 시에만.

## 5. 오프라인 및 빠른 실행

- **오프라인 캡처 보장**: Firestore 오프라인 persistence 활성화. 오프라인 저장은 로컬에 즉시 기록, 온라인 복귀 시 자동 동기화. AI 분류 제안은 동기화 이후 처리. 덤프 화면은 Service Worker로 프리캐시하여 비행기 모드에서도 열린다.
- **PWA shortcut**: Manifest `shortcuts`에 "새 덤프" 등록 — 홈 화면 아이콘 롱프레스로 덤프 화면 직행(커서 포커스 상태).
- **사이드 버튼 빠른실행 (기기 설정 안내)**: 갤럭시 Z 폴드 7 기준, 설정 > 유용한 기능 > 사이드 버튼 > 두 번 누르기 > 앱 실행에서 설치된 PWA 지정 (One UI 버전에 따라 메뉴명 상이할 수 있음). 설치 가이드 문서에 포함.

## 10. 검토 후 제외한 기능 (기록용)

## 11. 참고사항

- Claude API 모델명은 소스 내 설정 파일 한 곳에서만 관리한다 (기본: claude-sonnet-4-6).
- 심리 마스터 요약본은 최초 1회 생성 필수, 이후 #심리 기록 누적 시 수동 갱신 (기존 기획 유지).
- 네이버 블로그 아카이빙 데이터는 별도 정규화 스크립트로 Firestore에 일괄 업로드 예정 (본 개발과 별개 작업, 기존 기획 유지).
