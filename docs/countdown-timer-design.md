# 남은 시각이 표시되는 타이머 — 설계 문서 (MVP)

> 타겟 시각까지 남은 시간을 "양(면적)"으로 보여주고, 그 시간에 해야 할 일을 함께 띄워주는 단일 페이지 타이머.
> 주 사용자: 주의력 저하가 있는 아이들, 루틴을 지키기 어려운 사람, 지각이 잦은 사람.

---

## 0. 설계 원칙 (왜 이렇게 만드는가)

1. **숫자보다 면적.** ADHD 아동은 "17분 남았음"이라는 숫자보다, 줄어드는 색칠된 부채꼴(Time Timer 방식)을 훨씬 직관적으로 인지한다.
   → 아날로그 시계 위에 **현재 시각~타겟 시각 구간을 채운 부채꼴**을 그리고, 시간이 흐르면 부채꼴이 실시간으로 줄어든다.
2. **임박성은 색+움직임으로.** 남은 비율에 따라 부채꼴/배경 색이 초록 → 노랑 → 주황 → 빨강으로 바뀌고, 마지막 구간에서는 맥박(pulse) 애니메이션이 켜진다.
3. **한 화면, 조작 3번 이내.** 타겟 시각 선택 → 할 일 선택 → 시작. 설정 화면 없이 홈에서 전부 해결.
4. **기존 리포 패턴 준수.** 빌드 없음, 서버 없음, 단일 HTML 파일, localStorage 저장, 한국어, Gaegu 폰트 계열의 아이 친화적 톤. (`index.html` 카피바라 앱과 동일한 관례)

## 1. 데이터 구조 설계

서버가 없으므로 전부 브라우저 `localStorage` 하나의 키(`family-timer-v1`)에 JSON으로 저장한다.

```js
// ── 런타임 전역 상태 (ST) ─────────────────────────────
const DEFAULT_STATE = {
  // 실행 중인 타이머 (없으면 null)
  session: {
    targetAt: 1760340600000,   // 타겟 시각 (epoch ms) — "HH:MM"이 아닌 절대시각으로 저장해
                               //   새로고침/재방문해도 정확히 이어짐
    startedAt: 1760338800000,  // 시작 시각 (epoch ms) — 부채꼴의 "전체 크기" 기준
    taskId: 't3',              // 이 시간에 할 일 (presets 참조, 없으면 null)
    done: false                // 타겟 도달 후 "다 했어요!" 확인 여부
  },

  // 할 일 프리셋 — 부모가 미리 만들어 두고 아이는 고르기만 함
  presets: [
    { id: 't1', emoji: '🪥', label: '양치하고 세수하기' },
    { id: 't2', emoji: '🎒', label: '가방 싸고 신발 신기' },
    { id: 't3', emoji: '📚', label: '숙제 한 장 하기' }
  ],
  _seq: 4,                     // preset id 발급용 시퀀스

  settings: {
    sound: true,               // 종료 알림음 (Web Audio 합성음 — 오디오 파일 불필요)
    theme: 'sky'               // 배경 테마 (아이 취향 선택지)
  }
};
```

**핵심 결정 사항**

| 결정 | 이유 |
|---|---|
| 타겟을 `epoch ms` 절대시각으로 저장 | "남은 초"를 저장하면 새로고침 시 시간이 얼어버림. 절대시각이면 탭을 껐다 켜도 정확 |
| `startedAt`을 함께 저장 | 부채꼴 크기 = (target−now)/(target−started). 진행률 계산에 필수 |
| 할 일은 세션당 1개 | MVP는 "지금 이 시간엔 이것 하나"에 집중. 다단계 루틴(연속 슬롯)은 v2 |
| 파생값은 저장하지 않음 | 남은 시간·긴급도·색상은 매 tick마다 `Date.now()`로 계산 (단일 진실 원천) |

**파생값 (저장 안 함, 매초 계산)**

```js
remainMs  = max(0, session.targetAt - Date.now())
totalMs   = session.targetAt - session.startedAt
ratio     = remainMs / totalMs                     // 1 → 0
urgency   = ratio > .5 ? 'calm'                    // 🟢 초록: 여유
          : ratio > .25 ? 'aware'                  // 🟡 노랑: 인지
          : ratio > .1  ? 'hurry'                  // 🟠 주황: 서두름 + 은은한 pulse
          :               'critical'               // 🔴 빨강: 강한 pulse + 배경 점멸
```

## 2. 프로젝트 구조 (File Tree)

리포 관례(앱 1개 = HTML 파일 1개)를 따른다. 새 폴더/빌드 도구 없음.

```
family/
├── index.html                      # (기존) 카피바라 숙제 탐험대 — 건드리지 않음
├── timer.html                      # ★ 신규: 이 앱 전체 (HTML+CSS+JS 단일 파일)
└── docs/
    └── countdown-timer-design.md   # 이 문서
```

`timer.html` 내부 섹션 구성 (기존 index.html과 동일한 주석 구획 스타일):

```
timer.html
├── <style>            키프레임(pulse, 색 전환), 리셋 CSS
└── <script>
    ├── Constants      THEMES, URGENCY(색·문구·애니메이션 매핑), DEFAULT_STATE
    ├── State          load()/save() — localStorage 'family-timer-v1'
    ├── Time utils     remainMs/ratio/urgency 계산, mm:ss 포맷
    ├── Clock renderer SVG 아날로그 시계 — 큰 숫자 12개, 시침/분침/초침,
    │                  남은 시간 부채꼴(SVG path, 매초 갱신)
    ├── Screens        renderHome(시계+할일카드+남은시간), renderSetup(타겟·할일 선택),
    │                  renderDone(완료 축하 오버레이), renderPresets(할 일 관리)
    └── Tick loop      requestAnimationFrame 기반 1초 tick → 계산 → render
```

## 3. 단계별 개발 로드맵

각 단계가 끝날 때마다 **브라우저에서 열어 바로 확인 가능한 동작물**이 나온다.

### Step 1 — 시계 본체 (반나절)
- SVG 대형 아날로그 시계: 화면을 꽉 채우는 원, **선명하고 큰 숫자 1~12**, 시침·분침·초침 실시간 구동.
- 검증: 실제 시계와 시각 일치, 태블릿 가로/세로 모두 잘림 없음.

### Step 2 — 카운트다운 부채꼴 + 타겟 설정 (반나절)
- "몇 시까지?" 선택 UI: 자주 쓰는 프리셋 버튼(+10분/+30분/8:30/9:00 등) + 직접 선택.
- 현재 시각→타겟 시각 부채꼴을 시계 위에 오버레이, 매초 줄어듦.
- 남은 시간 디지털 표기(mm:ss)를 시계 아래 크게 병기.
- localStorage 저장 → 새로고침해도 이어짐.
- 검증: 12시간 넘는 타겟 방지(최대 12시간), 자정 넘김 처리.

### Step 3 — 임박성 연출 (반나절)
- urgency 4단계 색 전환(부채꼴+배경+숫자), hurry부터 pulse, critical에서 강한 점멸.
- 타겟 도달: 화면 전체 알림 + Web Audio 합성 알림음(첫 터치 후 활성화되는 브라우저 정책 대응) + "다 했어요!" 축하 오버레이(폭죽 애니메이션).

### Step 4 — 할 일 표시 (반나절)
- 시계 위/아래에 **큰 이모지 + 할 일 카드** 상시 표시 ("🪥 양치하고 세수하기").
- 프리셋 관리(추가/삭제/이모지 선택) — 부모용 간단 편집 화면.
- 타이머 시작 시 할 일 선택 단계 삽입 (건너뛰기 가능).

### Step 5 — 마감 품질 (반나절)
- 전체화면 버튼(태블릿 거치 대비), Screen Wake Lock API로 화면 꺼짐 방지.
- 소리 on/off, 테마 선택, 여러 기기 크기 반응형 점검.
- 완료 후 배포: GitHub Pages 등 정적 호스팅에 `timer.html` 그대로 업로드하면 끝.

**총 예상: 2~3일.** Step 1~3만으로도 "쓸 수 있는 물건"이 나오고, Step 4가 MVP 완성선.
